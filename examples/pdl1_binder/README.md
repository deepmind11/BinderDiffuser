# PD-L1 binder design example

A worked example of motif-scaffolded binder design against the IgV domain of
PD-L1 (programmed death-ligand 1, gene `CD274`).

## Why PD-L1?

PD-L1 is the canonical immune checkpoint target: blockade of the PD-1/PD-L1
axis underpins pembrolizumab (Keytruda), atezolizumab (Tecentriq), and
durvalumab (Imfinzi). It is also one of the most structurally well-characterized
binder design targets, with crystal structures of:

- PD-L1 alone (PDB `5C3T`)
- PD-L1 in complex with PD-1 (PDB `4ZQK`)
- PD-L1 in complex with antibody Fv (PDB `5O45`)
- PD-L1 in complex with peptide and small-molecule inhibitors (multiple)

This means we have a known binding face, several published de novo binders to
sanity-check against (Cao et al. *Nature* 2022 design pipelines), and clean
reference geometry for the motif we want to scaffold.

## Motif choice

The motif used here is residues 56-65 of PD-L1 (chain B in `5O45`), a
contiguous loop on the PD-1-binding face. Holding this loop's geometry fixed
during diffusion forces RFdiffusion to grow scaffold *around* the correct
epitope, biasing the entire generated cohort toward the productive face of
the protein.

## Files

| File | Purpose |
|------|---------|
| `config.yaml` | Pipeline configuration (target, motif, diffusion, MPNN, AF, filters) |
| `target/pdl1_5o45.pdb` | Target chain extracted from PDB 5O45 |
| `runs/` | Generated by `binderdiffuser run config.yaml` |

## Reproducing

```bash
# Preview the contigs without GPU
binderdiffuser show-contigs examples/pdl1_binder/config.yaml -n 5

# Full pipeline (GPU required for RFdiffusion + ColabFold)
binderdiffuser run examples/pdl1_binder/config.yaml

# Re-emit figures from a previous run
binderdiffuser report examples/pdl1_binder/runs/v1
```

## Note on the target PDB

`pdl1_5o45.pdb` is a stripped fragment of PDB 5O45 retaining the PD-L1 IgV
domain only (the antibody chain is removed). For the canonical full structure
download from RCSB:

```bash
wget https://files.rcsb.org/download/5O45.pdb -O target/5o45_full.pdb
```
